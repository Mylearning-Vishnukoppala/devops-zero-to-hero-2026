# Day 02: Software Development Life Cycle (SDLC)

## What is SDLC?
SDLC is the standard set of processes used in the software industry to design, develop, test, and deploy high-quality software products.

## The Standard SDLC Phases
- **Planning:** Defining requirements and goals.
- **Design:** Creating architectural blueprints.
- **Development:** Writing the application code.
- **Testing:** Validating code quality and functional requirements.
- **Deployment:** Releasing the application to the end users.
- **Monitoring/Maintenance:** Tracking performance and stability post-release.

## Waterfall vs. DevOps
- **Waterfall:** A linear, sequential approach. Errors are expensive and slow to fix because they are discovered late.
- **DevOps:** A cyclical approach (The Infinity Loop). By automating testing and deployment, we catch errors earlier and deliver updates continuously.

### How DevOps Engineers Transform the SDLC:
As a DevOps engineer, your core value add is focusing on the **Build, Test, and Deploy** phases. You improve the delivery process by ensuring these three stages occur in a fully automated, predictable, and repeatable way. 
- **Efficiency through Automation:** By replacing fragile manual handoffs with automation scripts (pipelines), you reduce human error, ensure consistent environment parity, and enable rapid, reliable code delivery.
